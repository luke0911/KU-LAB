## 슬라이드 7: OkHttpClient 문제 - 개념 설명
```
제목: OkHttpClient란?

[중앙 다이어그램]
앱 ←→ OkHttpClient ←→ 인터넷 ←→ 서버
     (우체부 역할)

역할:
📡 서버에 데이터 전송
📥 서버로부터 데이터 받기
🔒 암호화 통신 (HTTPS)
⏱️ 타임아웃 관리

생성 비용:
• 메모리: 5-10MB
• 시간: 수십 밀리초
• 연결 풀, SSL 설정, 스레드 생성
```

**레이아웃**: 중앙 다이어그램 + 설명

## 슬라이드 8: OkHttpClient 문제 - 기존 코드
```
제목: ❌ 기존 방식: 매번 새로 만듦

코드:
┌─────────────────────────────────┐
│ private fun performLogin(...) { │
│   val client = OkHttpClient()   │ ← 5-10MB 할당!
│   val gson = Gson()             │ ← 또 할당!
│   // 로그인 요청...              │
│ }                               │
└─────────────────────────────────┘

문제점:
❌ 로그인 5번 시도 = 25-50MB 낭비
❌ 연결 재사용 불가 (매번 100ms 소요)
❌ 가비지 컬렉터 부담 증가
```

## 슬라이드 9: OkHttpClient 문제 - 비유
```
제목: 🏭 비유: 공장을 계속 짓고 부순다

❌ 나쁜 방식:
제품 1개 만들기
→ 공장 짓기 (비용 큼)
→ 제품 만들기
→ 공장 부수기

제품 2개 만들기
→ 공장 또 짓기 (비용 큼)
→ 제품 만들기
→ 공장 또 부수기

✅ 좋은 방식:
공장 1번만 짓기
→ 제품 1개 만들기
→ 제품 2개 만들기
→ 제품 3개 만들기
→ 계속 재사용!
```

## 슬라이드 10: 개선 방법 - OkHttpClient 싱글톤
```

코드:
┌─────────────────────────────────────┐
│ object ApiClient {                  │
│   val timeoutClient by lazy {       │
│     OkHttpClient.Builder()          │
│       .connectTimeout(10, SECONDS)  │
│       .build()                      │
│   }                                 │
│ }                                   │
│                                     │
│ // 사용:                             │
│ ApiClient.timeoutClient.newCall()   │ ← 재사용!
└─────────────────────────────────────┘

효과:
✅ 메모리 5-10MB만 사용 (1개)
✅ 연결 재사용 (10ms로 단축)
✅ 성능 향상
```

## 슬라이드 12: SharedPreferences 중복 문제
```
제목: ❌ 문제: 서랍을 두 번 열고 닫음

기존 코드:
1. 서랍 열기 (10ms)
2. 4개 항목 저장
3. 서랍 닫기 (20ms) ← 파일 쓰기
4. 서랍 다시 열기 (10ms)
5. 2개 항목 저장
6. 서랍 다시 닫기 (20ms) ← 또 파일 쓰기

총: 60ms + 파일 쓰기 2번

개선 후:
1. 서랍 열기 (10ms)
2. 6개 항목 한 번에 저장
3. 서랍 닫기 (20ms) ← 파일 쓰기 1번

총: 30ms + 파일 쓰기 1번
→ 2배 빠름! 배터리 50% 절감!
```

**레이아웃**: Before/After 비교
**애니메이션**: 서랍 여닫는 동작
